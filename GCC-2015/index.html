Galaxy Docker container
Motivation
  • finding an appropriate Galaxy instance is hard
  • self-assembled Galaxy instances for everyone
  • moving software to the data
  • reproducible training instances
  • Isolation between Galaxy and the OS
  • better and faster testing
Galaxy Docker Container

docker run -p 8080:80 bgruening/galaxy-stable

Container?
Layers :: nested tupperware
Ingredients
  • Galaxy (stable, dev)
  • powered by ansible
  • FTP Server
  • Webserver
  • Scheduler
  • Galaxy Reports
  • Tool Shed install script
  • BioJS install script
  • Interactive Environments
Process control

docker run -it -e "NONUSE=proftp,reports" bgruening/galaxy-stable
